# 蒙面狼人杀 - 游戏创作思路

## 游戏名称
**蒙面狼人杀**

## 人数配置
根据搜索资料，狼人杀经典局为12人标准配置：
- 4个狼人
- 4个神民（预言家、女巫、猎人、守卫）
- 4个平民

## 游戏机制
1. **最少玩家要求**：开一局最少需要1个真人玩家
2. **AI数字人填充**：当真人玩家不足12人时，剩余名额由AI数字人自动填充
3. **AI数字人属性**：
   - 性格：随机生成（如开朗、内敛、逻辑型、直觉型等）
   - 阅历：随机生成（新手、普通、资深等）
   - 智商：随机生成（偏向普通真人水平，避免过高或过低）
4. **角色数字代号**：每个角色分配唯一数字代号（1-12）
5. **游戏流程**：
   - 身份分配
   - 夜晚行动（狼人杀人、预言家验人、女巫用药、守卫守护）
   - 白天发言
   - 投票放逐
   - 重复直至某一方获胜
6. **评分系统**：
   - 游戏结束后，系统对每位玩家的整局表现进行综合打分
   - 打分维度包括：
     - 逻辑推理能力
     - 发言质量
     - 团队贡献
     - 战略决策
     - 获胜加成
   - 每个回合都会产生打分明细，包括：
     - 夜间行动决策
     - 白天发言内容质量
     - 投票选择合理性
     - 信息传递有效性
   - 最终评分将展示给所有玩家，并提供详细的评分分析
7. **真人猜测环节**：
   - 游戏结束后，真人玩家需要猜测其他角色是真人还是AI数字人
   - 系统记录猜测结果，并计算猜测成功率
   - 猜测成功率单独记录在玩家的本局游戏数据中
   - 猜测成功率将作为玩家综合能力的参考指标之一

## 特色亮点
1. **AI数字人模拟**：每个AI数字人都有独特的性格和行为模式，模拟真实玩家
2. **蒙面机制**：玩家以蒙面形象出现，增加神秘感
3. **智能平衡**：系统根据AI数字人的表现自动调整难度，确保游戏体验
4. **数据统计**：记录玩家胜率、常用角色等数据，帮助玩家提升水平
5. **轻量级浏览器运行**：无需下载安装，随时随地可玩
6. **URL分享机制**：一局游戏开始前，分享唯一URL给任何真人，真人即可加入游戏
7. **纯文字式样**：界面简洁，纯文字操作，适合上班期间摸鱼玩
8. **低资源占用**：运行流畅，不会占用过多系统资源，不易被察觉

## 技术实现
1. **AI行为模拟**：基于性格、阅历、智商参数生成AI决策
2. **自然语言处理**：实现AI数字人的发言逻辑
3. **游戏平衡算法**：确保不同水平玩家都能获得良好体验
4. **独立Session通道**：每个AI数字人使用独立的session通道与大模型交流，防止被大模型认为是同一个人
   - 为每个AI数字人创建唯一的身份标识
   - 维护独立的对话历史和上下文
   - 确保AI数字人之间的行为和发言风格差异明显
5. **AI数字人池**：
   - 建立一个AI数字人池，每个数字人有独立的人格soul.md文件
   - 记录每个AI数字人的游戏历史、性格特征和能力水平
   - 游戏一局完成后，AI数字人根据游戏表现缓慢进化游戏能力和个人性格
   - 模拟真人学习模式，进化速度缓慢，符合人类学习曲线
   - 实现遗忘机制，AI数字人长时间不参与游戏后，能力会逐渐减退
   - 确保AI数字人的能力水平与真人玩家相当，保持游戏平衡性

## 未来扩展
1. **更多角色**：添加白狼王、白痴、丘比特等角色
2. **自定义规则**：允许玩家调整人数和角色配置
3. **社交功能**：添加好友系统、语音聊天等
4. **赛事系统**：组织线上比赛，增加游戏竞技性
5. **游戏模式多样化**：
   - 不同人数的游戏模式（6人局、9人局、12人局）
   - 快速模式（10-15分钟一局）和标准模式（30-45分钟一局）
   - 主题模式（节日主题、特殊规则主题）
6. **AI数字人个性化**：
   - 为AI数字人添加职业背景（教师、程序员、医生等）
   - 赋予AI数字人独特的口头禅和发言风格
   - 模拟不同年龄段的AI数字人，增加多样性
7. **游戏记录与回放**：
   - 保存每局游戏的完整记录
   - 提供游戏回放功能，支持快进、慢放和暂停
   - 允许玩家分享精彩对局片段
8. **成就系统**：
   - 设计多样化的成就徽章（如"狼人猎人"、"预言家大师"等）
   - 设立排行榜，展示玩家的胜率、评分等数据
   - 提供阶段性目标，鼓励玩家持续参与
9. **语言支持**：
   - 支持多语言切换，扩大游戏受众
   - 提供文本翻译功能，方便国际玩家交流
10. **安全措施**：
    - 实现防作弊系统，检测异常行为
    - 提供举报机制，处理恶意玩家
    - 保护玩家隐私，匿名参与游戏
11. **性能优化**：
    - 实现资源预加载，减少游戏加载时间
    - 采用代码分割，优化首屏加载速度
    - 支持离线模式，在网络不稳定时仍可游戏

## 创新思路

   - 让AI数字人从每局游戏中学习，不断改进行为模式
   - 基于玩家反馈调整AI数字人的表现
   - 实现AI数字人之间的互动学习，提升整体智能水平
2. **社区驱动内容**：
   - 允许玩家创建自定义角色和规则
   - 建立社区投票机制，选出最受欢迎的自定义内容
   - 定期举办社区活动，增强玩家粘性
3. **跨平台同步**：
   - 实现账号系统，支持多设备登录
   - 游戏进度和数据实时同步
   - 提供Web端和移动端的一致体验
4. **沉浸式体验**：
   - 添加音效增强沉浸感
   - 设计不同场景的背景描述，提升游戏氛围
   - 实现角色表情和动作的文字描述，增加画面感
5. **教育价值**：
   - 设计逻辑推理训练模式，帮助玩家提升思维能力
   - 提供游戏技巧和策略指南
   - 鼓励玩家分析和总结游戏经验

## 可玩性分析
### 优势
1. **低门槛参与**：最少只需1个真人玩家，适合不同场景
2. **AI数字人填充**：解决了传统狼人杀需要多人参与的问题
3. **独特的蒙面机制**：增加游戏神秘感和趣味性
4. **评分系统**：提供客观反馈，激励玩家提升
5. **真人猜测环节**：增加额外的游戏乐趣和挑战性
6. **轻量级浏览器运行**：无需下载安装，随时随地可玩
7. **URL分享机制**：方便邀请好友加入，提升社交性

### 挑战
1. **AI行为模拟**：需要确保AI数字人表现自然，避免机械感
2. **游戏平衡**：不同人数和AI水平组合下的平衡调整
3. **浏览器性能**：确保在不同设备和浏览器上流畅运行
4. **实时通信**：需要解决浏览器端的实时数据同步问题

### 建议
1. **渐进式开发**：先实现核心功能，再逐步添加扩展内容
2. **用户反馈**：收集玩家反馈，持续优化AI表现和游戏体验
3. **性能优化**：采用轻量级前端框架，确保游戏运行流畅
4. **安全措施**：防止恶意用户破坏游戏平衡

## 浏览器实现方案
1. **技术栈选择**：
   - 前端：HTML5 + CSS3 + JavaScript
   - 框架：Vue（根据开发团队偏好）
   - 通信：WebSocket实现实时交互
   - 后端：Java/Python（根据开发团队偏好）

2. **URL分享机制**：
   - 每局游戏生成唯一URL
   - 通过URL参数传递游戏ID和邀请信息
   - 支持短链接生成，方便分享

3. **数据存储**：
   - 游戏状态存储在服务器
   - 玩家数据本地存储，提升加载速度
   - 云端备份重要游戏数据

4. **纯文字界面实现**：
   - 简洁的文本界面，减少视觉元素
   - 响应式设计，适配不同屏幕尺寸
   - 键盘快捷键支持，操作更便捷
   - 低色彩方案，减少视觉冲击力

5. **摸鱼友好设计**：
   - 支持最小化到后台运行
   - 游戏通知采用低调的文本提示
   - 页面标题可自定义，伪装成工作相关页面
   - 支持快捷键快速切换到工作页面

## 开发参考
### 目标用户群体
- **年龄段**：不限
- **玩家类型**：兼顾休闲玩家和硬核狼人杀爱好者

### 开发优先级
- **核心功能**：基础游戏能力（身份分配、夜晚行动、白天发言、投票放逐、AI数字人填充、评分系统、真人猜测环节）
- **后期迭代**：社交功能（好友系统、语音聊天等）、赛事系统、更多角色、自定义规则

### 技术资源
- **开发团队**：独立一人开发
- **技术栈**：Java/Python（后端）、Vue（前端）
- **服务器资源**：有

### 移动端支持
- **优先级**：考虑移动端浏览器体验，手机浏览器优先
- **原生APP**：不计划开发

### 盈利模式
- **付费功能/广告**：暂时不考虑
- **虚拟物品/会员系统**：后期再考虑成本回收

### 游戏平衡性
- **人数配置平衡**：不需要针对不同人数进行平衡调整
- **AI难度调节**：暂时不需要，数字人能力每局随机生成
