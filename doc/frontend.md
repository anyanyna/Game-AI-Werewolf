# 前端界面模块详细设计文档

## 1. 模块概述

前端界面模块负责实现游戏的用户界面，包括纯文字界面、多端适配、游戏流程界面和摸鱼友好设计等核心功能。该模块确保游戏在不同平台上都能提供良好的用户体验。

## 2. 功能需求

### 2.1 纯文字界面
- 简洁的文本操作界面
- 低资源占用，适合上班期间摸鱼玩
- 响应式设计，适配不同屏幕尺寸

### 2.2 多端适配
- Web端：响应式设计，适配不同浏览器
- iOS/Android端：原生体验，适配不同屏幕尺寸
- 操作方式：Web端支持键盘快捷键和鼠标操作，移动端支持触摸操作和手势

### 2.3 游戏流程界面
- 身份分配界面：展示角色信息（蒙面状态）
- 夜晚行动界面：角色专属操作界面
- 白天发言界面：发言顺序、计时、内容展示
- 投票放逐界面：投票目标选择、结果展示

### 2.4 摸鱼友好设计
- 最小化功能：支持后台运行
- 通知机制：低调的文本提示
- 页面伪装：可自定义页面标题，伪装成工作页面
- 快速切换：快捷键快速切换到工作页面

### 2.5 URL分享机制
- 游戏唯一URL生成
- 短链接生成功能
- 通过URL加入游戏

## 3. 技术栈

### 3.1 核心技术
- **框架**：React + React Native (或 Flutter)
- **状态管理**：React Context 或 Redux
- **路由**：React Router
- **通信**：socket.io-client
- **样式**：Tailwind CSS 或 Styled Components
- **构建工具**：Webpack 或 Vite

### 3.2 第三方库
- **UI组件**：Ant Design 或 Material UI
- **图标**：React Icons
- **动画**：Framer Motion
- **表单**：Formik + Yup
- **工具库**：Lodash, Moment.js

## 4. 组件设计

### 4.1 核心组件

#### GameContainer
- **功能**：游戏主容器，管理游戏状态和布局
- **属性**：
  - `gameId`：游戏ID
  - `gameState`：游戏状态
  - `currentPhase`：当前阶段
- **子组件**：
  - GameHeader
  - GameContent
  - GameFooter

#### GameHeader
- **功能**：显示游戏状态、当前阶段、剩余时间
- **属性**：
  - `gameStatus`：游戏状态
  - `currentPhase`：当前阶段
  - `remainingTime`：剩余时间
- **功能点**：
  - 显示游戏标题
  - 显示当前阶段
  - 显示剩余时间
  - 提供设置按钮

#### GameContent
- **功能**：显示游戏主要内容，根据当前阶段切换不同界面
- **属性**：
  - `currentPhase`：当前阶段
  - `gameState`：游戏状态
  - `players`：玩家信息
- **子组件**：
  - IdentityAssignment
  - NightAction
  - DaySpeak
  - Voting
  - GameResult

#### GameFooter
- **功能**：提供操作按钮和输入框
- **属性**：
  - `currentPhase`：当前阶段
  - `canAct`：是否可以操作
- **功能点**：
  - 显示操作按钮
  - 提供输入框
  - 显示发言顺序

### 4.2 游戏流程组件

#### IdentityAssignment
- **功能**：显示角色分配结果（蒙面状态）
- **属性**：
  - `role`：角色
  - `roleCode`：角色数字代号
- **功能点**：
  - 显示角色信息（蒙面状态）
  - 提供确认按钮

#### NightAction
- **功能**：夜晚行动界面，根据角色显示不同操作
- **属性**：
  - `role`：角色
  - `players`：玩家列表
- **子组件**：
  - WolfAction
  - SeerAction
  - WitchAction
  - GuardAction

#### DaySpeak
- **功能**：白天发言界面
- **属性**：
  - `currentSpeaker`：当前发言者
  - `speakers`：发言顺序
  - `speechContent`：发言内容
  - `remainingTime`：发言剩余时间
- **功能点**：
  - 显示发言顺序
  - 显示当前发言者
  - 显示发言内容
  - 显示发言计时器

#### Voting
- **功能**：投票放逐界面
- **属性**：
  - `players`：玩家列表
  - `votes`：投票结果
- **功能点**：
  - 显示投票目标列表
  - 记录投票选择
  - 显示投票结果

#### GameResult
- **功能**：游戏结束界面，显示获胜方和评分
- **属性**：
  - `winner`：获胜方
  - `scores`：玩家评分
- **功能点**：
  - 显示获胜方
  - 显示玩家评分
  - 提供重新开始按钮

### 4.3 摸鱼友好组件

#### FishModeToggle
- **功能**：切换摸鱼模式
- **属性**：
  - `isFishMode`：是否为摸鱼模式
- **功能点**：
  - 切换摸鱼模式
  - 自定义页面标题

#### Notification
- **功能**：显示游戏通知
- **属性**：
  - `message`：通知消息
  - `type`：通知类型
- **功能点**：
  - 显示低调的文本通知
  - 自动消失

#### QuickSwitch
- **功能**：快速切换到工作页面
- **属性**：
  - `shortcutKey`：快捷键
- **功能点**：
  - 监听快捷键
  - 快速切换到工作页面

## 6. 状态管理

### 6.1 全局状态
- **游戏状态**：准备中、进行中、已结束
- **当前阶段**：准备、夜晚、白天、结束
- **玩家信息**：角色、状态、分数
- **游戏设置**：摸鱼模式、页面标题

### 6.2 局部状态
- **输入内容**：发言内容、投票选择
- **UI状态**：模态框显示、加载状态
- **动画状态**：过渡动画、微交互

### 6.3 状态管理方案
- **React Context**：管理全局状态
- **useState**：管理局部状态
- **useReducer**：管理复杂状态逻辑

## 7. 多端适配

### 7.1 Web端适配
- **响应式设计**：使用媒体查询和弹性布局

- **键盘快捷键**：支持常用操作的快捷键

- **触摸操作**：支持触摸、滑动等手势

### 7.3 适配策略
- **组件复用**：使用跨平台组件
- **条件渲染**：根据平台显示不同内容
- **样式适配**：使用平台特定的样式
- **性能优化**：根据平台特性进行优化

## 8. 通信设计

### 8.1 WebSocket通信
- **连接管理**：建立和维护WebSocket连接
- **消息处理**：处理游戏状态更新、玩家操作等消息
- **断线重连**：实现断线重连机制
- **消息队列**：处理网络延迟和消息顺序

### 8.2 API调用
- **游戏管理**：创建游戏、加入游戏、开始游戏
- **游戏操作**：夜晚行动、白天发言、投票放逐
- **用户管理**：登录、注册、个人信息

### 8.3 数据同步
- **实时同步**：确保所有玩家看到相同的游戏状态
- **状态恢复**：处理断线重连后的状态恢复
- **冲突处理**：解决并发操作冲突

## 9. 性能优化

### 9.1 前端优化
- **代码分割**：使用React.lazy和Suspense
- **资源预加载**：预加载关键资源
- **缓存策略**：合理使用浏览器缓存
- **渲染优化**：使用React.memo和useCallback
- **状态管理优化**：避免不必要的状态更新

### 9.2 网络优化
- **WebSocket优化**：减少消息大小，使用压缩
- **API优化**：减少API调用，使用批量请求
- **缓存策略**：缓存游戏数据，减少网络请求

### 9.3 多端性能优化
- **Web端**：优化DOM操作，减少重排重绘
- **移动端**：减少内存使用，优化动画性能
- **跨平台**：使用平台特定的优化策略
